# practic
## Прототип генератора безопасного тестового HTTP-трафика.

Асинхронная консольная утилита на Python для проведения нагрузочного тестирования локальных веб-интерфейсов, эмуляции сценариев поведения пользователей и валидации механизмов защиты стенда (Rate Limiting, WAF, Kill Switch). Инструмент разработан специально для использования в изолированных учебных и исследовательских контурах и снабжен встроенной жесткой системой пре-валидации безопасности.


### Обязательные ограничения безопасности:
- Запуск разрешен исключительно на localhost, интерфейс loopback (127.0.0.1) и домены, явно внесенные в белый список.
- Поддерживаются только протоколы HTTP и HTTPS. Категорически запрещены UDP, raw sockets, распределенный запуск и генерация деструктивного трафика.
- Если параметры не заданы в сценарии, система инициализирует тест с безопасными значениями: 2 RPS и 2 виртуальных клиента.
- Максимально допустимая интенсивность в учебной версии — 20 RPS и 20 виртуальных клиентов. Любая попытка указать значения выше (например, 40 RPS) блокируется на этапе синтаксического анализа.
- Автоматический переход по HTTP-перенаправлениям запрещен. Адрес назначения и DNS-записи повторно валидируются перед каждым новым соединением и после каждого редиректа. Если домен пытается увести трафик во внешнюю сеть, утилита экстренно останавливается.

### Основные директории и файлы:
#### app/:
- validate.py - Модуль безопасности и пре-валидации
- execute.py - Асинхронный исполнитель сценариев и планировщик
- metrics.py - Подсистема сбора метрик и асинхронного логирования
- main.py - Главная точка входа и CLI-интерфейс

#### sandbox/: Содержит конфигурационные файлы для развёртывания локального тестового стенда в изолированном контуре через докер.

#### scenarios/: Содержит все тестируемые YAML-сценарии.

#### tests/:
- test_kill_switch.py - Тесты для проверки срабатывания kill switch
- test_security.py - Тесты на прохождение валидации для validate.py

#### dummy_server.py 
- Базовый сервер-заглушка
#### dummy_server_protected.py 
- Локальный сервер с эмуляцией Rate Limiting
#### compare.py 
- Утилита дифференциального анализа и сравнения отчетов с выводом таблицы.


### Формат запуска:
В одном терминале тестирование с клиентами, в другом - сервер (в итоговой версии - сервер в докере).

#### Доступные команды: 
validate, dry-run, run, compare, stop

'''

python app/main.py <команда> scenarios/<сценарий.yaml>   #для validate, dry-run и run

python app/main.py <команда> <файл_1.json> <файл_2.json>   #для compare

python app/main.py <команда>   #для stop в другом терминале

'''
